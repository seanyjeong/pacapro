# MAX LINK 독립 정시 성적 조회

## 목표

MAX LINK 가족 회원은 PACA 교육원 원장이 정시엔진에 로그인하거나 지점을
연결한 적이 없어도, 본인에게 연결된 학생의 정시 성적을 조회할 수 있어야 한다.

## 사용자 흐름

1. MAX LINK가 가족 세션에서 확인한 `academy_id`와 PACA 학생 ID로 조회한다.
2. PACA는 학생이 해당 교육원 소속인지 먼저 검증한다.
3. PACA는 정시엔진의 실제 지점 목록과 교육원명을 비교해 우선 지점을 찾는다.
4. 지점명이 명확하지 않으면 정시 학생 ID 또는 이름·학교·학년의 유일 일치로
   지점을 자동 탐색한다.
5. 일치 학생과 성적이 있으면 안전한 성적 필드만 반환한다.
6. 학생은 일치하지만 성적이 없거나 학생을 찾지 못하면 `matched=false`를
   반환하며, 원장 연동을 요구하지 않는다.

## API 계약

- `GET /paca/jungsi/family-scores/{student_id}`
- 인증: `X-API-Key: <MAXLINK_READ_API_KEY>`
- 필수 쿼리: `academy_id`
- 선택 쿼리: `year`(기본 2027), `exam`(현재 기본 모의고사)
- PACA의 `academy_settings.jungsiLink`는 읽지 않는다.
- 성공 응답은 `academyId`, `branchName`, `matched`, `student`, `scores`만 포함한다.
- 전화번호, 원장 계정, 정시 토큰, 다른 학생 정보는 반환하지 않는다.

## 자동 식별 규칙

1. 교육원명과 정시 지점명을 브랜드·공백·`학원`·`교육원` 표현을 제거한 뒤
   비교한다.
2. 정확히 하나의 지점이 일치하면 그 지점만 조회한다.
3. 지점이 불명확하면 실제 정시 지점들을 제한된 동시성으로 읽고 전역에서
   학생을 찾는다.
4. 저장된 정시 학생 ID가 있으면 ID의 유일 일치를 우선한다.
5. ID가 없으면 이름과 학교를 반드시 비교하고, 학년이 양쪽에 있으면 학년도
   함께 일치해야 한다.
6. 둘 이상이 일치하면 어떤 학생 데이터도 반환하지 않는다.

## 보안 경계

- MAX LINK 전용 키는 지정된 가족 조회 GET 경로에만 사용할 수 있다.
- 같은 키로 POST, PUT, PATCH, DELETE를 호출할 수 없다.
- 모든 PACA 쿼리는 `academy_id + student_id`로 범위를 제한한다.
- 정시엔진 호출은 PACA 서버 내부의 서비스 토큰으로만 수행한다.
- PACA·정시엔진 DB에는 쓰지 않는다.

## 수용 기준

- 원장 `jungsiLink`가 없어도 가족 성적 조회가 성공한다.
- 가족 조회 중 `academy_settings` 쿼리가 발생하지 않는다.
- 교육원명이 명확하면 해당 지점만 정시엔진에서 조회한다.
- 지점명이 불명확해도 학생의 강한 유일 일치가 있으면 조회한다.
- 동명이인 또는 교육원 불일치 시 성적을 반환하지 않는다.
- MAX LINK는 `/jungsi/status`를 호출하지 않는다.
- 성적 미제출은 `academic.state=missing`, 원천 장애는 `unavailable`로 구분한다.
- 기존 원장용 정시 연결·성적 조회 동작은 회귀하지 않는다.

- PACA API: 무연동 성공, 타 교육원 차단, 전용 키 GET 허용·쓰기 차단.
- MAX LINK 계약: 단일 GET, 범위 쿼리, 안전한 응답 투영, 상태 구분.
- 회귀: PACA Jest 전체, MAX LINK pytest·ruff·mypy, 가족 포털 Vitest·Playwright.
