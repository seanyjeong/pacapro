# P-ACA 서비스 소개서

## 토스플레이스 연동용 서비스 개요

---

## 1. 서비스 개요

### 1.1 P-ACA란?

**P-ACA** (Physical Academy Comprehensive Administration)는 체대입시 전문 학원을 위한 종합 관리 시스템입니다.

| 항목 | 내용 |
|------|------|
| **서비스명** | P-ACA (파카) |
| **풀네임** | Physical Academy Comprehensive Administration |
| **서비스 URL** | https://pacapro.vercel.app |
| **API URL** | https://chejump.com/paca |
| **타겟 고객** | 체대입시 학원 |
| **운영 시작** | 2024년 |

### 1.2 서비스 목적

체대입시 학원의 **학생 관리**, **학원비 관리**, **스케줄 관리**, **강사 급여 관리** 등 학원 운영에 필요한 모든 기능을 하나의 시스템에서 제공합니다.

---

## 2. 주요 기능

### 2.1 학생 관리
- 학생 등록/수정/조회
- 학생 상태 관리 (재원/휴원/퇴원/졸업/체험)
- 학년 자동 진급 (매년 3월 1일)
- 학생 정보 암호화 저장

### 2.2 학원비 관리
- 월 수강료 자동 청구 (매월 1일)
- 납부 기록 관리
- 미납/부분납 추적
- 할인 적용 (정률/정액)
- 일할 계산 지원

### 2.3 스케줄 관리
- 시간대별 수업 배정
- 강사별/학생별 스케줄 조회
- 출석 체크

### 2.4 강사 급여 관리
- 수업 기반 급여 자동 계산
- 급여 명세서 생성

### 2.5 알림 서비스
- 학원비 납부 안내 알림톡
- 미납자 알림톡
- 체험수업 안내 알림톡
- 상담 예약 알림

---

## 3. 토스플레이스 연동 목적

### 3.1 연동 배경

현재 학원비 수납은 다음과 같은 방식으로 진행됩니다:

```
[기존 방식]
1. 학부모가 계좌이체 또는 현금 납부
2. 학원 직원이 P-ACA에 수동으로 납부 기록
3. 입금 확인 지연, 수동 입력 오류 발생 가능
```

### 3.2 연동 후 개선

```
[토스 연동 후]
1. 학원 직원이 토스 프론트에서 미납 학생 선택
2. 토스 터미널로 카드/NFC/현금 결제
3. 결제 완료 → P-ACA 자동 반영
4. 영수증 자동 출력
```

### 3.3 기대 효과

| 항목 | 개선 효과 |
|------|----------|
| **수납 편의성** | 카드/NFC/현금 결제 즉시 처리 |
| **자동화** | 수동 입력 불필요, 실시간 동기화 |
| **정확성** | 입력 오류 제거, 결제 이력 자동 기록 |
| **투명성** | 토스 영수증 발급, 결제 내역 추적 |

## 4. 연동 범위

### 4.1 P-ACA → 토스 프론트 (플러그인)

| 기능 | 설명 |
|------|------|
| 미납자 목록 조회 | 해당 학원의 미납/부분납 학생 목록 표시 |
| 학생 검색 | 이름/학번으로 학생 검색 |
| 결제 금액 확인 | 청구 금액, 남은 금액 표시 |

### 4.2 토스 → P-ACA (콜백)

| 기능 | 설명 |
|------|------|
| 결제 완료 알림 | 결제 성공 시 P-ACA API 호출 |
| 자동 매칭 | 주문번호로 결제 건 자동 매칭 |
| 납부 기록 | student_payments 테이블 자동 업데이트 |

## 5. 기술 스택

### 5.1 프론트엔드
- **프레임워크**: Next.js 15
- **스타일링**: TailwindCSS
- **호스팅**: Vercel

### 5.2 백엔드
- **프레임워크**: Express.js (Node.js)
- **데이터베이스**: MySQL 8.0
- **호스팅**: 자체 서버 (Linux)

### 5.3 보안
- **인증**: JWT (24시간 만료)
- **암호화**: AES-256-GCM (민감 정보)
- **통신**: HTTPS (TLS 1.2+)
- **비밀번호**: bcrypt 해싱
