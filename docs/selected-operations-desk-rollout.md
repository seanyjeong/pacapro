# PACA Selected UI Direction — Operations Desk Rollout

## 선택 방향

PACA 메인 UI는 **Other A · Operations Desk**를 기준으로 개발한다. 운영자가 PC에서 학생 검색, 상담, 출결, 결제, 후속관리를 반복하는 흐름에 가장 잘 맞고, 기존 어드민 구조를 크게 깨지 않아 단계적 이식이 가능하다.

## 고정할 골격

- 56px 아이콘 레일: 메뉴는 좁게, 현재 위치는 색상과 아이콘 상태로만 표시한다.
- 56px 상단 커맨드바: 브레드크럼, 통합 검색, 지점 선택, 오늘 날짜, 핵심 액션을 둔다.
- 좌측 360px 업무 큐: 오늘 상담, 미납, 출결 이상, 신규 등록처럼 당장 처리할 일을 시간순으로 보여준다.
- 메인 KPI 스트립: 6개 이하의 운영 숫자를 낮은 높이로 배치한다. 큰 영웅형 카드로 만들지 않는다.
- 메인 테이블: sticky header, 작은 상태 배지, 행 단위 액션을 기본 패턴으로 삼는다.
- 빈 상태와 오류 상태: 한국어 문장으로 다음 행동을 안내하고, 서버 코드나 개발 용어를 노출하지 않는다.

## 페이지별로 바꿀 수 있는 것

- 테이블 컬럼과 행 액션은 페이지 업무에 맞게 바꾼다.
- 좌측 업무 큐의 항목은 대시보드, 상담, 출결, 결제, 일정마다 우선순위를 다르게 둔다.
- 상단 primary 액션은 페이지당 하나만 유지한다.
- 현장 태블릿 업무는 같은 셸 안에 억지로 넣지 않고, 필요하면 별도 field mode로 분리한다.

## 1차 적용 순서

1. 대시보드 셸: 레일, 상단바, 업무 큐, KPI 스트립을 실제 라우트에 이식한다.
2. 학생 목록: 검색, 필터, 상태 배지, 행 액션, 빈/오류 상태를 먼저 맞춘다.
3. 학생 상세: 상담 메모, 출결 이력, 결제 내역을 같은 패널 규칙으로 정리한다.
4. 상담: 신규 상담 등록, 시간 선택, 지점별 설정 누락 안내를 자연어 오류로 처리한다.
5. 일정: 미등록 전환 학생이 일정에 남는지 확인하고, 원인별 안내와 동기화 상태를 분리한다.
6. 결제: 미납/분납/완납 배지와 저장 실패 UX를 통일한다.

## 페이지별 점검 체크리스트

- 데이터 로딩, 빈 상태, 저장 성공, 저장 실패, 권한 실패를 한 화면에서 확인한다.
- 프론트 타입과 백엔드 응답 필드가 일치하는지 확인한다.
- 브라우저 요청에서 인증 헤더, 쿠키, 네트워크 정책, 리다이렉트가 의도대로 동작하는지 확인한다.
- 오류 문구에는 서버 코드, 네트워크 정책명, 호출 흔적, 저장소 용어를 표시하지 않는다.
- 버튼 라벨은 사용자가 하려는 행동으로 쓴다. 예: "저장", "상담 예약", "출결 확인".
- 행 액션은 숨기지 않고, 적어도 메모/상세/더보기 진입은 항상 보이게 둔다.
- 1280px 데스크톱과 1024px 태블릿에서 표, 버튼, 배지가 겹치지 않는지 본다.

## 다음 개발 단위

첫 실제 이식은 PACA의 대시보드 또는 학생 목록 중 하나로 시작한다. 대시보드는 셸 안정성을 검증하기 좋고, 학생 목록은 실제 API 계약과 UX 문제를 빨리 드러낸다. 사용자가 매일 가장 많이 들어가는 화면을 첫 페이지로 잡는 것이 맞다.
